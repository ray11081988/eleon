# 工作流程详解

## 1. 导入会议记录

当用户提供会议记录时,按以下步骤处理:

### 步骤 1: 读取会议记录

- 支持多种格式: .txt, .md, .docx, .pdf
- 使用 Read 工具读取文件内容

### 步骤 2: 识别用户发言

- 检查会议记录中是否有发言人标注
- 常见格式包括:
  - "张三: 我认为..."
  - "张三说: ..."
  - "[张三] ..."
  - "张三\n我认为..."
- 如果没有明确标注,或有多个发言人,通过对话确认:
  - 列出会议中出现的所有发言人(如果能识别)
  - 询问用户: "这次会议中,你是哪位发言人?"
  - 或者展示几段典型发言,让用户确认哪些是他/她的

### 步骤 3: 提取用户发言

- 将用户的所有发言提取出来
- 保持原始语境和上下文
- 记录发言的时间顺序(如果有)

### 步骤 4: 提炼决策模型

- 分析用户发言中的决策模式
- 提取关键要素:
  - 核心价值观与原则
  - 思考框架与分析维度
  - 判断标准与评估指标
  - 风险偏好
  - 常用类比与思维模型
  - 决策模式
  - 语言风格
- 生成或更新决策模型文件

### 步骤 5: 保存记录

- 将原始会议记录保存到 `~/Eleon/meeting-records/`
- 将提取结果保存到 `~/Eleon/extraction-history/`
- 更新主决策模型文件 `~/Eleon/DECISION_MODEL.md`

## 2. 更新决策模型

当用户提供新的会议记录时:

### 步骤 1: 读取现有模型

- 从 `~/Eleon/DECISION_MODEL.md` 读取当前决策模型

### 步骤 2: 处理新会议记录

- 按照"导入会议记录"的流程处理新记录

### 步骤 3: 对比与合并

- 识别新的决策模式或思维方式
- 识别与现有模型的一致性或变化
- 合并到主模型中,保持连贯性

### 步骤 4: 记录变化

- 在提取历史中记录本次更新
- 注明新增或变化的要素

## 3. 应用决策模型

当用户提出新问题并要求用决策模型分析时:

### 步骤 1: 读取决策模型

- 从 `~/Eleon/DECISION_MODEL.md` 读取用户的决策模型

### 步骤 2: 理解问题

- 明确用户要分析的问题
- 识别问题的类型和领域

### 步骤 3: 应用模型分析

- 按照用户的思考框架来分析问题
- 使用用户的判断标准来评估
- 参考用户的风险偏好
- 借鉴用户常用的类比和思维模型

### 步骤 4: 呈现分析结果

- 明确说明: "以下分析基于你的个人决策模型"
- 说明使用了模型中的哪些要素
- 解释为什么这样分析符合你的思维方式
- 保持客观性,同时体现用户的决策风格

### 输出格式示例

```
【基于你的决策模型的分析】

根据你的决策模型,我从以下几个维度来分析这个问题:

1. [维度1 - 来自你的思考框架]
   - 分析内容...
   - 这符合你在[某次会议]中提到的[某个原则]

2. [维度2 - 来自你的判断标准]
   - 分析内容...

风险评估(基于你的风险偏好):
- ...

结论:
- ...

说明: 这个分析框架来自你在过去[N]次会议中的发言模式,特别是你在[具体场景]中展现的思维方式。
```
