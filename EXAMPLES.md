# 使用示例

## 示例 1: 导入第一份会议记录

**用户**: "我这里有一个产品评审会的会议记录,在 ~/Downloads/product_review.txt 里。能帮我导入一下吗?"

**助手操作流程**:

1. 读取文件 `~/Downloads/product_review.txt`
2. 分析会议记录格式,识别发言人
3. 如果有多个发言人,询问用户:
   ```
   我在会议记录中识别到以下发言人:
   - 张三
   - 李四
   - 王五

   请问你是哪位发言人?
   ```
4. 提取用户的所有发言
5. 分析决策模式,生成决策模型
6. 保存文件:
   - `~/Eleon/meeting-records/2026-03-31-product_review.txt`
   - `~/Eleon/extraction-history/2026-03-31-extract.md`
   - `~/Eleon/DECISION_MODEL.md`
7. 告知用户:
   ```
   已经从这次会议中提取了你的决策模式,保存在 ~/Eleon/DECISION_MODEL.md。

   主要识别出:
   - 3个核心原则
   - 4个思考维度
   - 2个常用类比

   你可以随时查看或更新这个模型。
   ```

## 示例 2: 更新决策模型

**用户**: "我又有一份投资讨论的会议记录,在 ~/Documents/investment.txt"

**助手操作流程**:

1. 读取现有决策模型 `~/Eleon/DECISION_MODEL.md`
2. 读取新会议记录 `~/Documents/investment.txt`
3. 识别用户发言
4. 提取新的决策模式
5. 对比分析:
   - 新模式与现有模式的一致性
   - 新增的思维方式
   - 可能的变化
6. 更新决策模型
7. 告知用户:
   ```
   已更新你的决策模型。

   本次更新:
   - 强化了"长期价值优先"的原则(与之前一致)
   - 新增了"财务健康度"评估维度
   - 补充了1个投资领域的类比案例

   决策模型现在基于 2 次会议记录。
   ```

## 示例 3: 应用决策模型分析问题

**用户**: "现在有个新的 SaaS 产品机会,用我的决策模型帮我分析一下。背景是..."

**助手操作流程**:

1. 读取决策模型 `~/Eleon/DECISION_MODEL.md`
2. 理解用户提出的问题
3. 按照用户的决策框架分析
4. 输出分析结果:

```
【基于你的决策模型的分析】

根据你的决策模型,我从以下几个维度来分析这个 SaaS 产品机会:

1. 市场与用户价值(来自你的核心原则"用户价值优先")
   - [分析内容...]
   - 这符合你在产品评审会中强调的"先验证真实需求"

2. 财务健康度(来自你的评估标准)
   - [分析内容...]
   - 参考你在投资讨论中提到的现金流评估方法

3. 长期可持续性(来自你的思考框架)
   - [分析内容...]

风险评估(基于你的风险偏好):
- 你倾向于在市场验证后再扩大投入
- 建议采用 MVP 方式先测试核心假设

结论:
- [具体建议...]

说明: 这个分析框架来自你在过去 2 次会议中的发言模式,
特别是你在评估新机会时展现的"先小后大"的思维方式。
```

## 示例 4: 处理发言人不明确的情况

**用户**: "帮我导入这份会议记录 ~/meeting.txt"

**助手**: (读取文件后发现没有明确的发言人标注)

```
我注意到这份会议记录没有明确标注发言人。

我看到记录中有这样几段内容:
1. "我认为我们应该优先考虑用户体验..."
2. "从技术角度来看,这个方案..."
3. "我建议我们先做个原型..."

请问:
- 这些都是你的发言吗?
- 还是只有部分是你的发言?

如果只有部分,请告诉我哪些段落是你说的。
```

## 常见问题

### Q: 如果会议记录很长怎么办?

A: 没问题,Eleon 会处理任意长度的会议记录。只提取你的发言部分进行分析。

### Q: 可以同时导入多份会议记录吗?

A: 建议一份一份导入,这样可以更好地追溯每个决策模式的来源。

### Q: 决策模型会自动更新吗?

A: 不会自动更新。每次导入新会议记录时,需要你确认是否更新模型。

### Q: 如果我的想法改变了怎么办?

A: 这很正常。当新会议记录显示你的思维方式有变化时,Eleon 会提示你,并询问是否更新模型。
